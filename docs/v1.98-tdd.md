                                                             ▼

## Component Designs

### 1. TaSurface (`src/core/ta_surface.ts`)

Assembles a TA snapshot per symbol. The LLM reads raw numbers and reasons about them — the quant does not interpret them.

**Snapshot schema:**
```typescript
interface TaSnapshot {
  symbol: string;
  price: number;
  priceVs24hHigh: number;      // % below 24h high (negative = below)
  priceVs24hLow: number;       // % above 24h low (positive = above)
  oiUsd: number;
  oiDelta1hPct: number;        // OI % change last 1h
  oiDelta4hPct: number;        // OI % change last 4h
  fundingRatePct: number;      // annualised funding rate
  volumeVs24hAvgPct: number;   // current 1h volume vs 24h hourly avg (%)
  priceVsEma20_1h: number;     // % above/below 20-period EMA on 1h candles
  trendBias: 'up' | 'down' | 'flat';  // derived from EMA20 slope across last 3 candles
  alertReason?: string;        // set if any threshold is crossed
}
```

**EMA computation specifics:**
- Source: `getCandles(symbol, '1h', 24)` — fetch 24 × 1h candles (existing `PriceService`)
- EMA20 requires ≥20 candles; if fewer returned, omit `priceVsEma20_1h` and set `trendBias = 'flat'`
- Slope: compare EMA20[last] vs EMA20[last-2]; threshold 0.1% for up/down classification
- Failure: if candle fetch throws, omit the snapshot entirely (do not block the rest)

**Fetch strategy:**
- All symbols fetched in parallel via `Promise.allSettled` — partial failures omit that symbol, never block the scan
- OI delta: compare current `oiUsd` to value from 1h ago (stored in market cache if available, else skip field)
- 4h OI delta: same approach with 4h cache window; omit if unavailable

**Alert thresholds (configurable, with defaults):**
```yaml
autonomy:
  ta:
    oiSpikePct: 8               # |oiDelta1hPct| > 8% → alert
    fundingExtremeAnnual: 50    # |fundingRatePct| > 50% → alert
    volumeSpikePct: 150         # volumeVs24hAvgPct > 150% → alert
```

Key-level alert (`priceKeyLevelPctWindow`) is deferred — requires a key-level store that doesn't exist yet.

**Key methods:**
- `TaSurface.computeAll(markets: string[]): Promise<TaSnapshot[]>` — parallel fetch, partial failures omitted
- `TaSurface.hasAlert(snapshot: TaSnapshot): boolean` — true if `alertReason` is set

### 2. LlmTradeOriginator (`src/core/llm_trade_originator.ts`)

**TradeProposal schema:**
```typescript
interface TradeProposal {
  symbol: string;
  side: 'long' | 'short';
  thesisText: string;               // narrative reason the trade makes sense now
  invalidationCondition: string;    // text description of what would prove thesis wrong
  invalidationPrice: number | null; // structured price level, if the LLM can specify one
  suggestedTtlMinutes: number;      // how long to give the thesis
  confidence: number;               // 0–1, LLM self-assessed
}
```

`invalidationPrice` is optional in the prompt schema (`number | undefined` in JSON). If the LLM provides it, the heartbeat uses it as a hard price stop (same as `position_exit_policy.invalidation_price`). If null/omitted, the exit consultant monitors via text-based thesis reasoning. This bridges the free-text ↔ structured enforcement gap.

**Input bundle:**
```typescript
interface OriginationInputBundle {
  book: BookEntry[];              // current open positions (PositionBook.getAll())
  taSnapshots: TaSnapshot[];      // TA surface across top-N markets
  marketContext: string;          // gatherMarketContext() output, max ~1000 chars
  recentEvents: string;           // recent events from events table, last 2h, max ~500 chars
  alertedSymbols: string[];       // symbols that triggered a TA alert this cycle
  performanceSummary?: string;    // phase 2: LLM's own originator track record
}
```

**Market context caching:**
`gatherMarketContext()` does web searches and is expensive. It is fetched at most once per 10 minutes regardless of how often the originator fires. Cache key = current 10-minute bucket (floor to 10m). Alert-triggered origination calls within the same 10-minute window reuse the cached context. This prevents N web searches per alert burst.

**System prompt principles:**
1. "Your default is NO TRADE. Return null unless a setup is clearly compelling."
2. "Maximum ONE proposal per call. If multiple setups look interesting, pick the best one."
3. "A valid proposal requires: a narrative thesis (why this trade makes sense RIGHT NOW), an invalidation condition (what would prove you wrong), and a TTL."
4. "You are looking for narrative + TA confluence — a story that the price structure confirms, not price structure alone."
5. "Null is the correct answer most of the time. Do not force a trade to fill the daily budget."
6. "If you can specify an exact price that would invalidate the thesis, include it as `invalidationPrice`. Otherwise omit it."

**Fallback chain:** main LLM (10s timeout) → trivial client (null-bias prompt, 5s timeout) → `null`. A null proposal is always safer than a bad proposal. No Telegram notify required on fallback — only log.

**minConfidence gate:** proposals with `confidence < config.autonomy.origination.minConfidence` (default `0.55`) are converted to null before reaching the entry gate. Logged with `entry_gate_verdict = 'confidence_gate'`.

**Symbol cooldown:** if the LLM proposed `symbol` in the last `cooldownMinutes` (default 30), suppress that symbol from the scan list passed to the next origination call. Applies on both rejected AND executed proposals — the idea was already acted on or considered; don't stack.

**Persistence:** each call (including nulls) written to `llm_trade_proposals` (see DB schema).

**Class interface:**
```typescript
class LlmTradeOriginator {
  constructor(
    private mainLlm: LlmClient,
    private fallbackLlm: LlmClient,
    private config: ThufirConfig,
  ) {}

  async propose(bundle: OriginationInputBundle): Promise<TradeProposal | null>
}
```

### 3. OriginationTrigger (`src/core/origination_trigger.ts`)

```typescript
type TriggerReason = 'cadence' | 'ta_alert' | 'event';

interface TriggerResult {
  fire: boolean;
  reason: TriggerReason;
  alertedSymbols: string[];
}

class OriginationTrigger {
  shouldFire(
    lastFiredMs: number,
    taSnapshots: TaSnapshot[],
    pendingEvents: Event[]
  ): TriggerResult
}
```

- **Cadence:** fire if `now - lastFiredMs >= cadenceMs` (default 15 min)
- **TA alert:** fire immediately if any snapshot has `hasAlert() === true`; return alerted symbols
- **Event:** fire immediately if any event in `pendingEvents` has `created_at > lastFiredMs`

Alert and event triggers bypass cadence. Cadence is the floor — LLM always gets at least one look per 15 min regardless.

### 4. AutonomousManager changes (`src/core/autonomous.ts`)

**New scan flow:**
```
1. PositionBook.refresh()
2. TaSurface.computeAll(topMarkets)                    ← always
3. OriginationTrigger.shouldFire()
4. if fire:
     a. assemble OriginationInputBundle
        (marketContext from cache if < 10 min old)
     b. LlmTradeOriginator.propose(bundle)
     c. if proposal != null:
          i.   LlmEntryGate.evaluate(proposal, markPrice)
          ii.  if approved/resized: quant sizing + risk → executor.execute()
          iii. exit policy: invalidationPrice (if set) else text-based TTL
5. if proposal == null AND trigger.reason == 'cadence' AND quantFallbackEnabled:
     runDiscovery() → expressions → LlmEntryGate.evaluate() per expression
   (quant fallback does NOT run on ta_alert or event triggers — only cadence)
```

**Gate mode for originator proposals:**
The entry gate prompt is passed a flag indicating this is a self-originated proposal. The gate system prompt adds: *"This proposal was generated by you. Your job here is adversarial — find reasons to reject it. Approve only if you cannot find a concrete flaw in the thesis or a conflict in the book."* This prevents the gate becoming a rubber stamp for self-generated ideas.

**Conflict guard:** existing `hasConflict` fast-path in entry gate handles opposite-side positions. `hasPosition` (same-side) warning still applies.

**Quant fallback scope:** quant path runs only on cadence ticks when LLM returned null. It never runs after a TA alert or event trigger — those are fast-path cycles where the LLM already looked and passed.

### 5. Prompt Design

**User message (assembled in `buildOriginationMessages`):**

```
## Open Positions
<symbol | side | ROE% | time held | thesis summary — or "(none)">

## Market Scan
<alerted symbols first, then remaining by OI rank>
BTC:  price=$X  OI_delta_1h=+3.2%  funding=+12% ann  vol=89% avg  trend=up   ema20_dist=+0.4%
HYPE: price=$Y  OI_delta_1h=-8.5%  funding=-22% ann  vol=210% avg trend=down  ema20_dist=-1.1%  [ALERT: OI spike, volume spike]
...

## Market Context
<gatherMarketContext() capped at 1000 chars — or "(not available)">

## Recent Events (last 2h)
<events table, capped at 500 chars — or "(none)">

## Instruction
Find ONE compelling trade setup, or return null.
A valid proposal requires all of:
  - symbol, side (long/short)
  - thesisText: WHY this trade makes sense right now (narrative + TA)
  - invalidationCondition: what would prove this thesis wrong
  - invalidationPrice: exact price level that invalidates (number, or omit if unclear)
  - suggestedTtlMinutes: how long to give this thesis
  - confidence: 0.0–1.0

Return ONLY valid JSON OR the literal string "null":
{"symbol":"...","side":"long"|"short","thesisText":"...","invalidationCondition":"...","invalidationPrice":number|null,"suggestedTtlMinutes":number,"confidence":number}
```

**Prompt discipline notes:**
- Alerted symbols appear at top of the market scan list
- Null framed as expected outcome, not failure
- `invalidationPrice` is explicitly optional with `number|null` — LLM can omit if no clean price level

## Phase 2: Feedback Loop (after 30–50 clean trades)

The `llm_trade_proposals` table accumulates all proposals (null and non-null) with outcome linkage via `trade_id`. Phase 2 adds:

1. **Nightly scoring job** — for each executed proposal, compute R-multiple outcome and write back to `llm_trade_proposals.outcome_r`
2. **Performance summary** — `summarizeOriginatorPerformance()`: null rate, win rate, avg confidence on winning vs losing proposals, per trigger reason
3. **Feed summary back to prompt** — `performanceSummary` in `OriginationInputBundle`; LLM reads its own track record as prior evidence

Intentionally out of scope for v1.98 phase 1. Schema designed now so data exists when phase 2 begins.

## Risks

| Risk | Mitigation |
|------|-----------|
| LLM overtrades (null rate < 50%) | Hard-framed null-default prompt; minConfidence gate (0.55); monitor null rate from `llm_trade_proposals` |
| Gate becomes rubber stamp for self-originated proposals | Adversarial gate mode: explicit prompt instruction to find reasons to reject self-generated ideas |
| LLM fixates on same symbols | Symbol cooldown (30 min) applied on both executed and rejected proposals |
| TA surface latency on large market list | `Promise.allSettled` parallel fetch; partial failures omit symbol, never block |
| Market context web search on every alert | Context cached with 10-min TTL; alert bursts reuse cached context |
| Quant fallback running on every alert tick | Quant path runs on cadence ticks only — not on ta_alert or event triggers |
| invalidationCondition is free text, heartbeat can't enforce it | `invalidationPrice` structured field bridges this; heartbeat uses it as hard stop if provided; exit consultant handles text-based thesis monitoring otherwise |
