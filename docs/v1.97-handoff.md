## What Is Missing

### Task 4 — `feat/v1.97-gateway-llm-wiring`

**The core problem:** The exit consultant and entry gate exist but the gateway doesn't wire real LLM clients into them. Nothing actually calls the LLM yet in production.

**Specific gaps:**

1. **`src/gateway/index.ts`** — `PositionHeartbeatService` is still constructed without an `exitConsultant`. Need to:
   - Instantiate `LlmExitConsultant(primaryAgent.llm, primaryAgent.infoLlm ?? primaryAgent.llm, heartbeatNotify, config)`
   - Pass it as `exitConsultant` in the heartbeat options
   - Guard with `config.positionHeartbeat?.llmExitConsult?.enabled !== false`

2. **`src/core/position_heartbeat.ts`** — `freshContext` is currently hardcoded as `''` in the exit consultant call. Needs to be replaced with:
   ```typescript
   const ctx = await gatherMarketContext(pos.symbol, config, { limit: 500 });
   freshContext = ctx.summary ?? '';
   ```
   Read `src/markets/context.ts` for the exact signature before implementing.

3. **`src/core/autonomous.ts`** — Entry gate `enabled` flag not yet respected. Need to check `(config.autonomy as any)?.llmEntryGate?.enabled !== false` before calling `gate.evaluate()`. If disabled, skip the gate entirely and proceed to execute.

4. **`config/default.yaml`** — Missing new config keys. Add:
   ```yaml
   # Under positionHeartbeat:
   positionHeartbeat:
     llmExitConsult:
       enabled: true
       firstConsultMinutes: 20
       cadenceMinutes: 20
       roeThresholds: [3, 7, 15]
       approachTtlMinutes: 15
       timeoutMs: 8000

   # Under autonomy:
   autonomy:
     llmEntryGate:
       enabled: true
       timeoutMs: 5000
       rejectOnBothFail: true
   ```

5. **`ThufirAgent` LLM accessors** — Check `src/core/agent.ts` to confirm `primaryAgent.llm` and `primaryAgent.infoLlm` are accessible from the gateway. May need getters if they're private.

### Task 5 — `feat/v1.97-integration-acceptance`

End-to-end tests that verify the full coherence pipeline works together:

- Autonomous scan with a conflicting position in DB → entry gate rejects → no order placed
- Position open 20+ min → exit consultant `shouldConsult()` fires → DB log entry written
- Main LLM unavailable → fallback fires → notify called
- `llmEntryGate.enabled: false` → gate skipped → trade executes as before
- `llmExitConsult.enabled: false` → heartbeat runs rules-only, no LLM calls
- Full regression: existing heartbeat trigger paths (time stop, invalidation, ROE close) still work

## Known Issues / Notes from Agent Reports

- Task 3 agent confirmed: `executePolicyClose(pos, 'llm_exit_consultant', ...)` is reused for LLM-triggered closes. The heartbeat has no dedicated reduce method — reduce replicates the `perp_place_order` reduce-only pattern directly.
- `lastConsultDecision` stores `roeAtConsult` so `shouldConsult()` can detect ROE threshold crossings on subsequent ticks — this is in-memory state that lives on `BookEntry` and is carried across refreshes.
- `PositionBook.refresh()` uses `listPaperPerpPositions()` — for live mode this returns `[]`. Live position support in the book is a known gap, lower priority since system is on paper.
