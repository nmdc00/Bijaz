# v1.98 Product Requirements Document
## LLM-Primary Trading: Narrative + TA Origination

**Release branch:** `release/v1.98`
**Status:** Planning
**Date:** 2026-03-20
**Depends on:** v1.97 shipped and stable; 30–50 clean gated trades recorded

---

## Problem Statement

v1.97 made the LLM a gatekeeper: the quant pipeline proposes candidates, the LLM approves or rejects them. This is better than no gate, but the quant's proposals are still built on a broken world model.

~483 pre-v1.97 trades show 0% win rate on `momentum_breakout` and `mean_reversion` — the two primary archetypes. These are not calibration errors. The quant classifies trades by signal pattern, but crypto moves on narrative: ETF decisions, funding extremes, protocol exploits, macro fear, influencer flows. Signal archetypes cannot reason about any of these. The quant is generating candidates the LLM should mostly reject — and the few it approves are already filtered through a broken selection bias.

The second problem is reactivity. The current scan is cadence-based (every ~300s). Markets move on TA confluences (key level tests, OI sweeps, funding flips) that have nothing to do with news. By the time a cadence-based scan picks up the setup, the entry is already suboptimal or gone.

---

## Goals

1. **LLM as originator:** The LLM proposes trade ideas based on narrative context and TA setup quality. The quant validates sizing and risk only — it does not classify or filter direction.
2. **TA as proactive signal:** The system surface TA data (key levels, OI delta, funding, volume, trend structure) to the LLM on every origination call — not as a trigger to fire, but as data to reason with. The LLM evaluates whether the TA setup is compelling, not the quant.
3. **Three trigger modes:** Origination fires on: (a) 15-min cadence; (b) TA alert (price crosses key level, OI spike, funding extreme, volume spike); (c) event detection (news catalyst detected). Cadence is the floor; alerts and events fire immediately.
4. **Default no-trade discipline:** The system prompt hard-constrains the LLM to return `null` by default. A trade proposal requires a thesis (narrative reason), an invalidation condition (what would make this wrong), and a TTL. No thesis → no trade.
5. **Quant as data supplier:** Discovery pipeline continues to run, producing signal data that the LLM reads. It does not produce executable trade candidates in the primary path. If the LLM returns null, the quant path may run as a fallback (configurable).

---

## Non-Goals

- Replacing the quant's sizing, leverage, and risk management logic — these remain fully deterministic.
- Real-time price prediction. The LLM reasons about narrative, TA context, and setup quality.
- Removing the v1.97 entry gate — all originator proposals still pass through the gate (self-review).
- Multi-agent orchestration. All LLM calls use the existing `createAgenticExecutorClient` pattern.

---

## User Stories

**As the system operator:**
- I want the system to propose trades when there is a compelling narrative + clean TA setup, not just when the quant detects a signal pattern.
- I want the system to be fast — TA alerts and event detections should fire origination immediately, not on the next cadence tick.
- I want the LLM to be disciplined — most scan cycles should produce no proposal. If null rate drops below 50%, something is wrong.
- I want every proposal logged with full reasoning, so I can review the LLM's decision quality over time.

**As the LLM agent (Thufir):**
- I want to receive TA data (OI delta, funding rate, volume, price structure, key levels) alongside market narrative so I can evaluate setup quality — not just narrative in isolation.
- I want to propose a trade only when I can articulate both why it should work and what would invalidate the thesis.
- I want my decisions recorded so the system can build a track record and eventually feed it back to me as prior evidence.

---

## Success Criteria

- LLM-originated trades account for >50% of executed trades within 2 weeks of shipping.
- Null-proposal rate >60% of scan cycles — confirms "default no-trade" discipline is holding.
- Win rate on LLM-originated trades measurably higher than quant-only baseline (target >30%).
- TA alert trigger fires within 60 seconds of threshold crossing (vs up to 300s on cadence).
- All proposals logged to DB with full thesis + invalidation condition for future feedback loop use.
- Full test suite passes (`pnpm test && pnpm typecheck`).

---

## Prerequisites

1. v1.97 shipped and stable — LLM entry gate and exit consultant running in production.
2. 30–50 clean v1.97-gated trades recorded — enough for the LLM's own track record to be meaningful once Phase 2 feedback loop is wired.
3. Phase 2 data reset complete — contaminated pre-v1.97 dataset truncated before v1.98 data collection begins.
