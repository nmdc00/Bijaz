# v2.1 Sprint Plan
## Extractable Learning Substrate

**Release branch:** `release-v2.1`  
**Status:** Draft execution plan  
**Date:** 2026-05-05

---

## Sprint Goal

Turn Thufir's improvement loop into a reusable learning framework by:

- separating comparable forecast learning from execution learning
- removing fake comparable semantics from perps
- normalizing learning cases
- making policy outputs explicit and auditable

---

## Task Map

### Task 1 — `feat/v2.1-learning-case-foundation`

**Owner area:**

- `src/memory/schema.sql`
- `src/memory/db.ts`
- `src/memory/learning_cases.ts`
- tests for schema + CRUD

**Acceptance:**

- canonical `learning_cases` table exists
- comparable/execution views exist
- typed memory APIs exist and pass tests

### Task 2 — `feat/v2.1-comparator-hygiene`

**Owner area:**

- `src/core/autonomous.ts`
- `src/memory/calibration.ts`
- `src/execution/modes/paper.ts`
- `src/execution/modes/webhook.ts`

**Acceptance:**

- perps stop writing fake comparable rows
- prediction-market comparable flow still works
- resolved outcomes update canonical comparable learning cases

### Task 3 — `feat/v2.1-execution-learning-engine`

**Owner area:**

- `src/core/adaptive_edge.ts`
- `src/core/execution_learning.ts`
- `src/core/perp_lifecycle.ts`
- `src/core/tool-executor.ts`

**Acceptance:**

- closed perps emit normalized execution-quality learning cases
- hierarchical backoff works
- adaptive edge can use the new learner

### Task 4 — `feat/v2.1-policy-audit-surface`

**Owner area:**

- `src/gateway/dashboard_api.ts`
- `dashboard/src/types.ts`
- `dashboard/src/App.tsx`

**Acceptance:**

- dashboard/API expose comparable vs execution counts
- exclusions by reason are visible
- active policy outputs are visible and typed
