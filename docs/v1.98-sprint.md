# v1.98 Sprint Plan
## LLM-Primary Trading: Narrative + TA Origination

**Release branch:** `release/v1.98`
**Status:** Planning
**Date:** 2026-03-20

---

## Sprint Goal

Replace the quant as the source of trade ideas. The LLM receives TA data, market context, and event signals, and decides whether there is a compelling trade setup — or returns null. The quant computes the data and manages sizing/risk. The LLM makes the directional call.

## Task Breakdown

### Task 1 — `feat/v1.98-ta-surface`
**TA data surface layer — structured market snapshots for LLM consumption**

- `src/core/ta_surface.ts` — `TaSurface` class
- `computeAll(markets: string[]): Promise<TaSnapshot[]>` — parallel fetch across top-N markets
- Per snapshot: price, 24h range position, OI delta (1h/4h), funding rate, volume vs avg, EMA-based trend bias
- `hasAlert(snapshot): boolean` — true if any configured threshold is crossed
- Config keys: `autonomy.ta.oiSpikePct`, `fundingExtremeAnnual`, `volumeSpikePct`, `priceKeyLevelPctWindow`
- Data sources: existing Hyperliquid market tools — no new API dependencies
- **Tests:** snapshot fields populated correctly from mock market data; alert detection fires on each threshold type; partial fetch failure omits symbol without blocking others

**Foundational — must land first.**

---

### Task 2 — `feat/v1.98-trade-originator`
**LlmTradeOriginator — LLM proposes trades from narrative + TA, or returns null**

- `src/core/llm_trade_originator.ts` — `LlmTradeOriginator` class
- `propose(bundle: OriginationInputBundle): Promise<TradeProposal | null>`
- System prompt: default-no-trade framing; requires thesis + invalidationCondition + TTL for a valid proposal
- JSON parse with Zod validation; literal `"null"` response returns `null` safely
- Fallback chain: main LLM (10s timeout) → trivial client → `null` (no notify, null is safe)
- `minConfidence` config gate: proposals below threshold become `null` before reaching entry gate
- `src/memory/llm_trade_proposals.ts` — `recordTradeProposal()` insert helper
- Schema migration: `llm_trade_proposals` table (see TDD)
- **Tests:** returns null when LLM emits `"null"` or `{"null":true}` variants; returns null when LLM unavailable; valid proposal passes Zod; confidence gate works; DB write verified; fallback does not call notify

**Depends on Task 1 (for TaSnapshot type).**

---

### Task 3 — `feat/v1.98-origination-trigger`
**OriginationTrigger — three-mode firing logic replacing cadence-only scan**

- `src/core/origination_trigger.ts` — `OriginationTrigger` class
- `shouldFire(lastFiredMs, taSnapshots, pendingEvents)` → `{ fire, reason, alertedSymbols }`
- Cadence mode: fires if `now - lastFiredMs >= cadenceMs` (default 15 min)
- TA alert mode: fires immediately if any snapshot `hasAlert()` is true; returns alerted symbol list
- Event mode: fires immediately if new event in DB since `lastFiredMs`
- Alert and event triggers bypass cadence — they fire on the current tick regardless of last cadence time
- Config key: `autonomy.origination.cadenceMinutes`
- **Tests:** cadence fires at correct interval; TA alert fires before cadence; event fires before cadence; alerted symbols list correct; multiple alerts in same tick = single fire

**Depends on Task 1 (for TaSnapshot type).**

---

### Task 4 — `feat/v1.98-autonomous-wiring`
**Wire originator into AutonomousManager scan loop**

- Modify `src/core/autonomous.ts` scan loop:
  1. `TaSurface.computeAll(topMarkets)` at scan start
  2. `OriginationTrigger.shouldFire()` to determine whether to call originator
  3. If fire: assemble `OriginationInputBundle` → `LlmTradeOriginator.propose()`
  4. If proposal: `LlmEntryGate.evaluate()` (v1.97) → quant sizing → execute
  5. Exit policy written from `proposal.invalidationCondition` + `proposal.suggestedTtlMinutes` (replacing generic TTL default)
  6. If null + `quantFallbackEnabled`: run existing discovery pipeline path
- Symbol cooldown: if LLM proposed a symbol in the last N min (configurable), suppress from scan list
- Config key: `autonomy.origination.quantFallbackEnabled` (default `true`)
- **Tests:** proposal → gate → execute path wired end-to-end; null → quant fallback path wired; symbol cooldown prevents duplicate proposals; quant path blocked when LLM has active position on symbol; exit policy TTL uses LLM's suggested TTL, not default

**Depends on Tasks 1, 2, 3; and on v1.97 components (LlmEntryGate, PositionBook).**

---

### Task 5 — `feat/v1.98-integration-acceptance`
**Integration and acceptance tests for full origination pipeline**

- E2E: TA alert detected → originator proposes → gate approves → order placed → proposal logged
- E2E: originator returns null → quant fallback runs → expression candidate goes through gate
- E2E: main LLM unavailable → originator returns null → quant fallback runs (no crash, no block)
- E2E: LLM proposes symbol that has an active position → gate rejects via conflict fast-path
- Null rate assertion: in mocked conditions with flat/neutral market data and no events, originator returns null >80% of the time (validates default-no-trade discipline in prompt)
- Regression: v1.97 heartbeat trigger paths (time stop, invalidation, exit consultant) unaffected
- `pnpm test && pnpm typecheck` fully green

**Depends on Tasks 1–4.**

## Risks

| Risk | Mitigation |
|------|-----------|
| LLM overtrades — null rate too low | Monitor null rate metric; if <50% in first week, tighten prompt or raise minConfidence threshold |
| LLM fixates on a few symbols | Symbol cooldown config; review proposal log after first 20 trades |
| TA surface fetch latency adds to scan time | Parallel fetch across markets; partial failures don't block scan |
| originator + entry gate = two sequential LLM calls per trade | Both have timeouts (10s + 5s); combined worst-case 15s — acceptable for a 15-min cadence |
| Quant fallback path undermines LLM-primary goal | Default-on initially to maintain trade flow; disable after 30+ LLM-originated trades validate quality |
