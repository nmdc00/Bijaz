# Release v1.95 Planner

Date: 2026-03-12

## Scope Guardrails

1. One feature branch per task, one PR per feature branch.
2. v1.95 ships the causal reasoning substrate, not a full commodities executor.
3. Historical case quality matters more than corpus size in this sprint.
4. Every task branch must include tests for newly introduced behavior.

## Test Coverage Requirements (Per Branch)

1. `feat/v1.95-event-schema-memory`
- Unit: schema migration and repo CRUD coverage.
- Integration: empty DB and migrated DB both load safely.

2. `feat/v1.95-event-extraction-core`
- Unit: dedupe/grouping and stable event-key generation.
- Integration: recent intel fixture collapses to expected event count.

3. `feat/v1.95-thought-generation-validation`
- Unit: JSON validator clamps/repairs/ rejects invalid outputs.
- Integration: material event fixture yields thought with mechanism, chain, assets, invalidation.

4. `feat/v1.95-forecast-outcome-loop`
- Unit: forecast status transitions and scheduled resolution behavior.
- Integration: forecast persists and later resolves into outcome record.

5. `feat/v1.95-domain-market-context`
- Unit: context interface adapters and fallback behavior.
- Integration: crypto adapter parity maintained with current functionality.

6. `feat/v1.95-proactive-loop-refactor`
- Unit: proactive refresh no longer fails solely due to absent perp-only evidence.
- Integration: non-crypto event question still gets fresh intel/web context.

7. `feat/v1.95-historical-casebase-seed`
- Unit: case schema validation for seed corpus.
- Integration: seed cases load and can be retrieved by mechanism/domain tags.

8. `feat/v1.95-event-tools-surface`
- Unit: tool DTO formatting and null handling.
- Integration: event/thought artifacts are visible in chat/mentat flows.

9. `feat/v1.95-integration-acceptance`
- Integration: intel -> event -> thought -> forecast chain.
- Regression: full test suite plus new event-pipeline acceptance fixtures.
