## 2. Conversations + Logs API

### New endpoints

#### `GET /api/conversations`

Returns session list, newest first.

```ts
type ConversationSession = {
  sessionId: string;
  messageCount: number;       // user + assistant only
  firstMessage: string;       // first user message, truncated to 120 chars
  startedAt: string;          // ISO, first message timestamp
  lastMessageAt: string;      // ISO, most recent message timestamp
};

type ConversationsListResponse = {
  sessions: ConversationSession[];
};
```

SQL:
```sql
SELECT
  session_id               AS sessionId,
  COUNT(*)                 AS messageCount,
  MIN(created_at)          AS startedAt,
  MAX(created_at)          AS lastMessageAt,
  (
    SELECT content FROM chat_messages cm2
    WHERE cm2.session_id = cm.session_id
      AND cm2.role = 'user'
    ORDER BY created_at ASC LIMIT 1
  )                        AS firstMessage
FROM chat_messages cm
WHERE role IN ('user', 'assistant')
GROUP BY session_id
ORDER BY MAX(created_at) DESC
```

#### `GET /api/conversations/:sessionId`

Returns the full thread for one session.

```ts
type ConversationMessage = {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  createdAt: string;
};

type ConversationThreadResponse = {
  sessionId: string;
  messages: ConversationMessage[];
};
```

SQL: `SELECT * FROM chat_messages WHERE session_id = ? AND role IN ('user', 'assistant') ORDER BY created_at ASC`

#### `GET /api/logs`

Returns decision audit entries and agent incidents, merged and sorted by time.

```ts
type ToolTraceEntry = {
  toolName: string;
  input: Record<string, unknown>;
  success: boolean;
};

type DecisionAuditEntry = {
  kind: 'decision';
  id: number;
  createdAt: string;
  source: string;
  sessionId: string | null;
  marketId: string | null;
  tradeAction: string | null;
  confidence: number | null;
  edge: number | null;
  criticApproved: boolean | null;
  toolCallCount: number;
  iterations: number;
  toolTrace: ToolTraceEntry[];
  planTrace: unknown;         // raw AgentPlan JSON
  notes: unknown;
};

type IncidentEntry = {
  kind: 'incident';
  id: number;
  createdAt: string;
  goal: string | null;
  toolName: string | null;
  error: string;
  blockerKind: string | null;
  details: unknown;
};

type LogsResponse = {
  entries: (DecisionAuditEntry | IncidentEntry)[];
  total: number;
};
```

Query params: `?limit=50&offset=0&kind=decision|incident|all`

Decisions: `SELECT * FROM decision_audit ORDER BY created_at DESC LIMIT ? OFFSET ?`
Incidents: `SELECT * FROM agent_incidents ORDER BY created_at DESC LIMIT ? OFFSET ?`

Merge in application layer, re-sort by `createdAt` DESC.

### TTL caching

Add a simple cache module `src/gateway/dashboard_cache.ts`:

```ts
type CacheEntry<T> = { data: T; expiresAt: number };

const store = new Map<string, CacheEntry<unknown>>();

export function cached<T>(key: string, ttlMs: number, fn: () => T): T {
  const entry = store.get(key);
  if (entry && Date.now() < entry.expiresAt) return entry.data as T;
  const data = fn();
  store.set(key, { data, expiresAt: Date.now() + ttlMs });
  return data;
}
```

Apply to all dashboard API handlers:

| Endpoint | TTL |
|---|---|
| `/api/dashboard` (equity, performance, trades) | 30s |
| `/api/dashboard` (positions, policy) | 5s |
| `/api/conversations` (session list) | 10s |
| `/api/conversations/:id` (thread) | 10s |
| `/api/logs` | 15s |

## 3. Conversations tab

### `ConversationsTab.tsx`

Two views: session list and thread. State: `selectedSessionId: string | null`.

**Session list:**
- Grouped by date (Today / Yesterday / older by week)
- Each row: first message preview (120 chars), message count, relative time
- Click → load thread

**Thread view:**
- Back button → session list
- Messages rendered as chat bubbles: user right-aligned, assistant left-aligned
- Assistant content rendered as markdown (use a lightweight renderer —
  `marked` or just handle `**bold**`, `\n`, code fences manually to avoid a
  heavy dependency)
- Timestamps shown on hover
- Scroll to bottom on load

**Empty state:** "No conversations yet." if sessions list is empty.

## 4. Logs tab

### `LogsTab.tsx`

Single feed of log entries, newest first. Paginated (load more button, 50 per page).

Filter bar: `All | Decisions | Incidents`

**Decision entry:**
```
[timestamp]  DECISION  market: BTC  action: open_long  edge: 4.2%  confidence: 71%
  ▶ Tool trace (click to expand)
      run_discovery        ✓
      perp_market_data     ✓
      perp_open_position   ✓   (3 tools, 2 iterations)
```

Expanded tool trace shows tool name, truncated input JSON (200 chars), success/fail badge.

**Incident entry:**
```
[timestamp]  INCIDENT  tool: perp_open_position  blocker: insufficient_margin
  error message text
```

**Empty state:** "No log entries yet."
