## 1. Bug fix: `_reflex` signal class

**File:** `src/discovery/expressions.ts:67–73`

**Current:**
```ts
const signalClass = hypothesis.id.includes('_revert')
  ? 'mean_reversion'
  : hypothesis.id.includes('_reflex')
    ? 'news_event'           // ← wrong
    : hypothesis.id.includes('_trend')
      ? 'momentum_breakout'
      : 'unknown';
```

**Fix:**
```ts
const signalClass = hypothesis.id.includes('_revert')
  ? 'mean_reversion'
  : hypothesis.id.includes('_reflex')
    ? 'liquidation_cascade'  // ← correct
    : hypothesis.id.includes('_trend')
      ? 'momentum_breakout'
      : 'unknown';
```

Impact: reflexivity expressions no longer route through `evaluateNewsEntryGate`.
They pass/fail based on `isSignalClassAllowedForRegime` (allowed in `trending` and
`high_vol_expansion`) and the edge threshold only.

Note: `liquidation_cascade` is currently blocked in `choppy` and
`low_vol_compression`. This is correct — reflexivity setups require elevated
volatility or trending conditions to unwind.

## 2. New module: `src/core/adaptive_edge.ts`

### Types

```ts
import type { ThufirConfig } from './config.js';
import type { PerpTradeJournalEntry } from '../memory/perp_trade_journal.js';

export type AdaptiveEdgeSegment = {
  signalClass: string;
  marketRegime: string;
  volatilityBucket: string;
  liquidityBucket: string;
};

export type AdaptiveEdgeResult = {
  edge: number;
  source: 'prior' | 'empirical' | 'blended';
  sampleCount: number;
  empiricalExpectancy: number | null;
  priorEdge: number;
  signalStrengthMultiplier: number;
};
```

### `resolveAdaptiveEdge`

**Signature:**
```ts
export function resolveAdaptiveEdge(
  config: ThufirConfig,
  journals: PerpTradeJournalEntry[],
  segment: AdaptiveEdgeSegment,
  signalStrength: number,  // 0–1, from cluster.confidence
): AdaptiveEdgeResult
```

**Algorithm:**

```
1. Read config:
     priorEdge        = autonomy.adaptiveEdge.priorEdge        ?? 0.015
     minSamples       = autonomy.adaptiveEdge.minSamples        ?? 10
     signalScaleFactor = autonomy.adaptiveEdge.signalScaleFactor ?? 0.5
     decayHalfLifeDays = autonomy.adaptiveEdge.decayHalfLifeDays ?? null (off)

2. Filter journals for segment:
     entries = journals where:
       - outcome === 'executed'
       - capturedR != null
       - signalClass === segment.signalClass
       - marketRegime === segment.marketRegime
       - volatilityBucket === segment.volatilityBucket
       - liquidityBucket === segment.liquidityBucket

3. Apply optional time decay (if decayHalfLifeDays != null):
     weight_i = 2 ^ (-(age_days_i / decayHalfLifeDays))
     Use weighted mean instead of simple mean

4. Compute empiricalExpectancy:
     If entries.length === 0: empiricalExpectancy = null
     Else: empiricalExpectancy = mean(entries.map(e => e.capturedR))

5. Blend:
     If entries.length < minSamples:
       blendedEdge = priorEdge          // pure prior
       source = 'prior'
     Else:
       weight = min(1, entries.length / (minSamples * 3))  // saturates at 3× minSamples
       blendedEdge = (1 - weight) * priorEdge + weight * empiricalExpectancy
       source = weight >= 1 ? 'empirical' : 'blended'

6. Apply signal strength multiplier:
     // signalStrength ∈ [0,1] → multiplier ∈ [1 - signalScaleFactor, 1 + signalScaleFactor]
     multiplier = 1 - signalScaleFactor + signalStrength * signalScaleFactor * 2
     edge = blendedEdge * multiplier

7. Floor at 0 (never return negative edge — that's the Sharpe gate's job):
     edge = max(0, edge)

8. Return AdaptiveEdgeResult
```

**Notes:**
- The floor at 0 means a signal class with negative empirical expectancy stops
  generating eligible expressions naturally (edge < minEdge) without an explicit blocker
- Negative expectancy segments are still visible in the journal for the Sharpe gate,
  which acts as a hard block after enough samples

## 4. Config keys

Add to `config/default.yaml` under `autonomy:`:

```yaml
autonomy:
  adaptiveEdge:
    enabled: true
    priorEdge: 0.015          # edge used before minSamples reached (1.5%)
    minSamples: 10            # samples needed per segment before empirical kicks in
    signalScaleFactor: 0.5    # how much discovery confidence modulates edge (0 = ignore, 1 = full scale)
    decayHalfLifeDays: null   # time decay for old trades (null = off)
```

Add to `src/core/config.ts` Zod schema under `autonomy`:

```ts
adaptiveEdge: z.object({
  enabled: z.boolean().optional(),
  priorEdge: z.number().min(0).max(1).optional(),
  minSamples: z.number().int().min(1).optional(),
  signalScaleFactor: z.number().min(0).max(1).optional(),
  decayHalfLifeDays: z.number().positive().nullable().optional(),
}).optional(),
```

## 7. Test requirements

### `tests/core/adaptive-edge.test.ts` (new)

| Test | Assertion |
|---|---|
| Returns priorEdge when no journal entries | `edge === priorEdge * multiplier` |
| Returns prior when entries < minSamples | `source === 'prior'` |
| Returns empirical when entries ≥ minSamples | `source === 'empirical' or 'blended'` |
| Positive capturedR history → edge > prior | Empirical pulls edge up |
| Negative capturedR history → edge floors at 0 | `edge === 0` |
| signalStrength=1.0 → max multiplier | `edge === blended * (1 + signalScaleFactor)` |
| signalStrength=0.0 → min multiplier | `edge === blended * (1 - signalScaleFactor)` |
| Blending saturates at 3× minSamples | `source === 'empirical'` at 30 samples (minSamples=10) |
| Segment filtering is exact match on all 4 keys | Entries from different regime not counted |
| `enabled: false` → legacy formula | `edge === confidence * 0.1` for non-neutral |

## 8. Full-universe market discovery

**Branch:** `feat/v1.93-full-universe-markets`

Two config changes that unlock the existing `selectDiscoveryMarkets` full-universe path:

| Key | Before | After |
|---|---|---|
| `hyperliquid.symbols` | `["BTC", "ETH"]` | `[]` |
| `paper.liveSymbolsAllowlist` | `['BTC', 'ETH']` | `[]` |

With `symbols: []`, `discoverySelection.fullUniverseWhenSymbolsEmpty: true` (already set)
causes `selectDiscoveryMarkets` to rank the entire Hyperliquid perp universe by OI +
volume + spread, returning the top `preselectLimit` (24) symbols per scan. All pass the
`liveSymbolsAllowlist` gate (empty = no restriction); paper mode default is unchanged.

No code changes required — the selection and gating infrastructure was built in v1.91.
