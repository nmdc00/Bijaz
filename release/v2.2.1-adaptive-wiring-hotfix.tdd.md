## Runtime Contract

### 1. Autonomous open / blocked path

Every `v2.2` `upsertTradeDossier()` call must pass:
- `dossier`
- `retrieval`
- `policyTrace`

The dedicated columns must match the same semantic object carried inside `dossier_payload`.

### 2. Executed open path

For every executed autonomous open:
- dossier status is `open`
- dossier version is `v2.2`
- retrieval payload persists both in:
  - `dossier.retrieval`
  - `trade_dossiers.retrieval_payload`
- policy trace persists both in:
  - `dossier.policy_trace`
  - `trade_dossiers.policy_trace_payload`
- similarity features are derived and upserted

### 3. Full-close path

For every full close:
- existing dossier is updated, not downgraded
- dossier version remains `v2.2`
- close section is populated
- review section is populated
- compact counterfactual summary remains in dossier JSON
- explicit counterfactual rows are written to `trade_counterfactuals`
- typed similarity features are refreshed from final close/review state
- `execution_quality` and `thesis_quality` remain linked to the same dossier id

### 4. Policy-adjustment path

For eligible closed trades:
- derive bounded policy-adjustment proposals from closed-trade evidence
- persist proposals to `trade_policy_adjustments`
- decision-time runtime loads active adjustments
- `applyAdaptiveDecisionEnforcement(...)` is used to adjust candidate size/leverage/verdict before the entry gate
- persisted policy traces include active adjustment ids

## Data Rules

### `trade_dossiers`

- `retrieval_payload` must be populated whenever `dossier.retrieval` is populated
- `policy_trace_payload` must be populated whenever `dossier.policy_trace` is populated
- close-path merge must not replace a `v2.2` dossier root with `v2.1`

### `trade_counterfactuals`

On every full close with enough evidence:
- persist `no_trade`
- persist `approved_size`
- persist other bounded records when inputs are present:
  - `full_size`
  - `delay_entry`
  - `ttl_exit`
  - `leverage_cap`

### `trade_similarity_features`

Feature derivation must be deterministic from dossier + review + learning-case context.

Minimum fields to populate when available:
- `signal_class`
- `trade_archetype`
- `market_regime`
- `volatility_bucket`
- `liquidity_bucket`
- `entry_trigger`
- `gate_verdict`
- `failure_mode`
- `success_driver`
- `thesis_verdict`
- `entry_quality`
- `sizing_quality`
- `session_bucket`

### `trade_policy_adjustments`

Activation rules:
- only persist adjustments when evidence count meets the bounded threshold already implied by `deriveTradePolicyAdjustments()`
- do not create hard policy reject authority from weak evidence
- persisted rows must be inspectable and reversible
