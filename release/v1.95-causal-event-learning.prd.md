# PRD: Causal Event Learning — v1.95

## Problem

Thufir can ingest news and search the web, but he does not yet convert fresh events into durable causal market reasoning artifacts. The current loop is still too crypto/perp-shaped and too ephemeral. It stores intel, not event-driven market thoughts that can be graded and reused.

If Thufir is meant to evolve from crypto trader to trader in general, he needs:
- normalized events,
- structured thoughts,
- explicit forecasts,
- later outcomes,
- and historical analog retrieval.

## Goal

Ship the first event-driven causal learning loop:

`intel -> event -> thought -> forecast -> outcome -> learning`

This release does not attempt full non-crypto execution. It ships the reasoning substrate, persistence layer, and the first historical casebase needed for general-trader expansion.

## Scope

1. Add event/thought/forecast/outcome memory objects.
2. Add event extraction from existing intel items.
3. Add structured thought generation with mechanism and causal chain.
4. Add forecast persistence and scheduled resolution hooks.
5. Add a historical casebase format and seed cases.
6. Refactor proactive reasoning interfaces so domain-aware market context is possible.

## Non-goals

1. Full commodity execution support.
2. Fine-tuning the LLM.
3. Massive automated historical backfill.
4. Production-grade commodity market adapters beyond scaffolding.

## Success Criteria

| Metric | Target |
|---|---|
| Recent intel can produce a normalized event | Required |
| Material event produces schema-valid thought artifact | Required |
| Thought includes mechanism + `causalChain` + impacted assets | Required |
| Forecast object is stored with explicit horizon + invalidation | Required |
| Historical case seed set logged and queryable from docs/spec | Required |
| Architecture no longer assumes perp tools are the only valid market context | Required |

## Risks

1. LLM thoughts may be plausible but weak unless validation is strict.
2. Event clustering can over-merge or under-merge.
3. Outcome scoring can be misleading if horizons are underspecified.
4. Commodity expansion still depends on market-data adapter selection.
