## Test Requirements

### Unit
- event key stability
- event deduplication
- thought JSON validation
- forecast schema validation
- outcome resolver status transitions

### Integration
- intel items -> event
- event -> thought
- thought -> forecast
- scheduled rehydrate of pending forecast checks
- proactive refresh works without requiring crypto-only signals
- macro/commodity question forces domain-specific intel/web retrieval
- geopolitical oil event links to impacted assets such as `CL`, `BRENTOIL`, or `GOLD` when appropriate
- retrieval miss is surfaced as an explicit source/extraction gap, not a false "no catalyst" conclusion

### Release Gate
- `pnpm run build`
- `pnpm vitest run`
- targeted event-pipeline tests green

### Required Failure-Mode Regression

Add a regression fixture for the exact class of miss seen in production:
- prompt asks whether Iran/Hormuz escalation could push oil higher
- fresh web/intel evidence exists
- system must not answer from generic crypto snapshot + BTC/ETH funding context alone
- output must either:
  - cite the oil-specific event and impacted assets, or
  - explicitly state that oil-specific retrieval/extraction failed
