# PRD: Dashboard v2 — v1.94

## Problem

The dashboard was built as an internal monitoring tool. It is now the first thing
anyone sees when they encounter Thufir. The current implementation has three problems
that make it a poor product showcase:

1. **Wrong stack.** CDN React with in-browser Babel transpilation is brittle and
   slow to render. One CDN outage and the page is blank. No TypeScript safety on the
   frontend. No real component structure.

2. **Stale data.** 30-second polling means positions and equity can be significantly
   out of date. For a live trading system this erodes trust.

3. **Missing the story.** The existing tabs show results (equity, trades, performance).
   They don't show *reasoning*. A visitor has no way to understand that Thufir is an AI
   that actively thinks, searches, analyses, and decides. The conversations and the
   decision audit trail are the most compelling parts of the system and they are
   completely invisible.

---

## Goal

Rebuild the dashboard as a product showcase that tells the full Thufir story:
what it trades, how it performs, what it says, and what it does under the hood.

---

## How it works

### Stack change

Replace CDN React + Babel with a proper Vite + React + TypeScript frontend. The build
output (`dashboard/dist/`) is served as static files by Express. The existing
`/api/dashboard` endpoint is unchanged. Two new endpoints are added.

Polling interval drops from 30s to 5s. The API response for live data (positions,
equity snapshot) is cached at 5s TTL to prevent SQLite hammering.

### Conversations tab

Lists all chat sessions grouped by date. Clicking a session opens the full thread.
Shows only `user` and `assistant` messages — system messages are hidden. Content is
plain prose (tool call results are already text-ified before storage in the current
LLM layer, so no stripping required).

This tab is read-only. No ability to send messages from the dashboard.

The goal is to let a visitor read a real conversation between the user and Thufir:
the analysis, the questions, the market calls. This is the product pitch.

### Logs tab

Shows Thufir's decision audit trail: every trade decision with its tool trace (which
tools ran, what inputs they received, whether they succeeded), the plan object, and
the confidence/edge scores that drove the decision.

Secondary sources: agent incidents (errors and blockers with classification) and
alert lifecycle events.

This tab is for the technically curious visitor who wants to see what's happening
beneath the conversation. It is not a raw log dump — entries are structured and
readable.

---

## Success criteria

| Metric | Target |
|---|---|
| Page load with no CDN dependency | Required |
| Position data freshness | ≤ 5s lag |
| Conversations tab renders real sessions | Required |
| Logs tab shows tool traces from `decision_audit` | Required |
| `pnpm build` produces a working static bundle | Required |
| Existing tab behaviour unchanged | Required |

---

## Non-goals

- Real-time WebSocket push (5s polling is sufficient for this use case)
- Authentication / access control (internal tool, not public-facing yet)
- Sending messages from the dashboard
- Backtesting or historical simulation views
- Mobile layout (desktop-first for now)
